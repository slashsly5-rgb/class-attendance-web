<task type="checkpoint:human-action">
  <name>Create GitHub repository and add remote</name>
  <action>Create GitHub repository and add remote (requires GitHub account)</action>
  <instructions>
**Option A: Using GitHub CLI (if available)**

If GitHub CLI is installed:
```bash
gh auth login
gh repo create class-attendance --private --source=. --remote=origin --push
```

This creates the repository, sets the remote, and pushes in one command.

**Option B: Using GitHub Web UI**

1. Visit https://github.com/new
2. Fill in repository details:
   - **Repository name**: class-attendance (or your preferred name)
   - **Visibility**: Private (recommended - contains project metadata)
   - **Initialize this repository**: Leave all checkboxes UNCHECKED (we already have code)
3. Click "Create repository"
4. Copy the HTTPS URL shown (looks like https://github.com/username/class-attendance.git)
5. In your terminal, run:
   ```bash
   git remote add origin https://github.com/username/class-attendance.git
   git branch -M main
   git push -u origin main
   ```

**Verify:**
- Refresh GitHub repository page - you should see all project files
- README.md, package.json, src/ directory visible

Why manual: GitHub repository creation requires GitHub account authentication. While `gh` CLI can automate this, it requires interactive OAuth login first.
  </instructions>

Type "done" when repository is created and code is pushed to GitHub.

<task type="checkpoint:human-action">
  <name>Connect GitHub repository to Vercel and configure environment variables</name>
  <action>Connect GitHub repository to Vercel and configure environment variables (requires Vercel account)</action>
  <instructions>
**Step 1: Create Vercel project**

1. Visit https://vercel.com and sign in (or create account)
2. Click "Add New..." → "Project"
3. Import your GitHub repository:
   - If first time: Click "Connect GitHub Account" and authorize Vercel
   - Find "class-attendance" in repository list
   - Click "Import"

**Step 2: Configure project settings**

On the configuration screen:
- **Framework Preset**: Next.js (should auto-detect)
- **Root Directory**: ./ (default, leave unchanged)
- **Build Command**: Leave default (`next build`)
- **Output Directory**: Leave default (`.next`)
- **Install Command**: Leave default (`npm install`)

**Step 3: Add environment variables**

Click "Environment Variables" section:

Add two variables:
1. **Name**: `NEXT_PUBLIC_SUPABASE_URL`
   - **Value**: [paste the Supabase URL from .env.local]
   - **Environments**: Check all three (Production, Preview, Development)

2. **Name**: `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - **Value**: [paste the anon key from .env.local]
   - **Environments**: Check all three (Production, Preview, Development)

**Step 4: Deploy**

Click "Deploy" button. Vercel will:
1. Clone repository
2. Install dependencies
3. Run `next build`
4. Deploy to production

Wait 2-3 minutes for initial deployment. You'll see build logs in real-time.

**Step 5: Get deployment URL**

Once deployment succeeds:
- Vercel shows "Congratulations!" with deployment URL
- URL format: https://class-attendance-xxxxx.vercel.app
- Click "Visit" to see deployed application

**Step 6: Verify automatic deployments**

In repository settings → Integrations → Vercel, verify:
- Production branch: main
- Automatic deployments: Enabled

Why manual: Vercel project creation requires account authentication and GitHub OAuth authorization. Environment variable configuration requires interactive UI (no headless alternative for initial setup).
  </instructions>
  <resume-signal>
Type "done" when Vercel project is deployed and accessible.

Provide the Vercel deployment URL (e.g., https://class-attendance-xxxxx.vercel.app)
  </resume-signal>
</task>
