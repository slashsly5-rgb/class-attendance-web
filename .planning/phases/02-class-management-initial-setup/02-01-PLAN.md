**File 1: src/lib/validations/class.ts**

Create Zod schema for class form validation following 02-RESEARCH.md Pattern 3:

```typescript
import { z } from 'zod'

export const classSchema = z.object({
  name: z.string().min(3, 'Class name must be at least 3 characters').max(100),
  degree_level: z.enum(['Bachelor', 'Master'], {
    errorMap: () => ({ message: 'Degree level must be Bachelor or Master' }),
  }),
  semester: z.string().min(1, 'Semester is required'),
  location_lat: z.coerce.number().min(-90).max(90).nullable(),
  location_lng: z.coerce.number().min(-180).max(180).nullable(),
  location_radius: z.coerce.number().int().min(10).max(500).default(50),
})

export type ClassFormData = z.infer<typeof classSchema>
```

Key details:
- Use z.coerce.number() for numeric fields from FormData (FormData.get() returns strings)
- location_lat/lng are nullable (optional during creation per 02-RESEARCH.md Open Question 3)
- location_radius defaults to 50 meters
- TypeScript types auto-inferred via z.infer

**File 2: src/lib/utils/code-generator.ts**

Implement nanoid-based code generator following 02-RESEARCH.md Pattern 4:

```typescript
import { customAlphabet } from 'nanoid'

// Custom alphabet: uppercase letters + numbers (no ambiguous chars like 0/O, 1/I)
const alphabet = '23456789ABCDEFGHJKLMNPQRSTUVWXYZ'
const nanoid = customAlphabet(alphabet, 8)

export function generateClassCode(): string {
  return nanoid() // Returns 8-char code like "A3K7M9XP"
}
```

Key details:
- 32-character alphabet (no 0, O, 1, I to avoid confusion)
- 8-character length (collision probability ~1% at 1M codes, negligible for 100 classes)
- Cryptographically secure randomness built into nanoid

Create src/lib/utils directory if it doesn't exist.
