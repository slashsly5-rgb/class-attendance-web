Create Server Actions file with 'use server' directive for analytics aggregation:

**Function 1: `getClassAnalyticsSummary(classId: string)`**
Purpose: Fetch summary metrics for dashboard cards
Returns: `{ classInfo, totalSessions, totalStudents, avgAttendance, studentsAtRisk }`

Query strategy (avoid N+1):
- Single query: JOIN classes -> enrollments -> students
- Separate query: Fetch all attendance_sessions for class
- Separate query: Fetch all attendance_records for those sessions (IN clause)
- Aggregate client-side: Calculate per-student attended/absent/late counts
- Calculate avgAttendance: Sum of (attended/totalSessions) across all students / totalStudents
- Count studentsAtRisk: students.filter(s => s.absent >= 2).length

Handle division by zero: `totalSessions > 0 ? calculation : 0`

**Function 2: `getClassAnalyticsDetail(classId: string)`**
Purpose: Fetch detailed per-student metrics for class analytics page
Returns: `{ classInfo, totalSessions, studentMetrics: Array<StudentMetric> }`

StudentMetric type:
```typescript
{
  id: string
  full_name: string
  student_id: string
  totalSessions: number
  attended: number
  absent: number
  late: number
  percentage: number
  healthStatus: 'excellent' | 'warning' | 'critical'
}
```

Same aggregation strategy as Summary, but include full student details.
Apply `getHealthStatus(percentage)` to each student.

**Function 3: `getAllClassesAnalytics()`**
Purpose: Fetch analytics for all classes in parallel for main dashboard
Returns: `Array<{ classId, summary }>`

Implementation:
```typescript
const classes = await getClasses()
const analyticsPromises = classes.map(c => getClassAnalyticsSummary(c.id))
const analytics = await Promise.all(analyticsPromises)
return classes.map((c, i) => ({ ...c, analytics: analytics[i] }))
```

**Error handling:** Wrap Supabase calls in try/catch, throw meaningful errors.

**Performance:** Profile query time. If >500ms, flag for future RPC optimization (Open Question 1 from research).

**Imports:** Import `createClient` from Supabase, `getHealthStatus` from threshold utils, `getClasses` from existing actions.

**Research patterns applied:** Pattern 1 (Server Component Data Aggregation), Pitfall 1 fix (N+1 avoidance), Pitfall 2 fix (division by zero).
