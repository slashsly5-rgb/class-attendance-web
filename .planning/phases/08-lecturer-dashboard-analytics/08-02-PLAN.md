  <files>src/components/features/analytics/ClassAnalyticsCard.tsx</files>
  <action>
Create client component ('use client') for displaying class analytics with visual health indicators:

**Props interface:**
```typescript
interface ClassAnalyticsCardProps {
  classData: {
    id: string
    name: string
    degree_level: string
    semester: string
    code: string
  }
  analytics: {
    totalSessions: number
    totalStudents: number
    avgAttendance: number
    studentsAtRisk: number
  }
}
```

**Component structure:**
- Use shadcn/ui Card component
- Apply conditional border-l-4 styling based on health status from `getHealthStatus(avgAttendance)`
- Use `getHealthColor()` for background tint and border color

**Layout (from research Pattern 3):**
```tsx
<Card className={cn(
  "border-l-4 hover:shadow-lg transition-shadow cursor-pointer",
  healthColor.border,
  healthColor.bg
)}>
  <CardHeader>
    <div className="flex justify-between items-start">
      <div>
        <CardTitle>{name}</CardTitle>
        <CardDescription>{degree_level} • {semester}</CardDescription>
      </div>
      <Badge variant={healthStatus === 'excellent' ? 'default' : healthStatus === 'warning' ? 'secondary' : 'destructive'}>
        {healthStatus.toUpperCase()}
      </Badge>
    </div>
  </CardHeader>
  <CardContent>
    <div className="grid grid-cols-2 gap-4">
      <MetricDisplay label="Avg Attendance" value={`${avgAttendance.toFixed(1)}%`} />
      <MetricDisplay label="Total Sessions" value={totalSessions} />
      <MetricDisplay label="Enrolled Students" value={totalStudents} />
      <MetricDisplay
        label="At Risk"
        value={studentsAtRisk}
        alert={studentsAtRisk > 0}
        className={studentsAtRisk > 0 ? "text-red-600 font-bold" : ""}
      />
    </div>
  </CardContent>
</Card>
```

**MetricDisplay helper component (inline):**
Small component showing label + value with optional alert styling.

**Imports:** Card, CardHeader, CardTitle, CardDescription, CardContent, Badge from shadcn/ui, cn from utils, getHealthStatus/getHealthColor from thresholds.

**Responsive:** Ensure touch-friendly on mobile (card min-height, adequate padding).

**Empty state handling:** Show "No data yet" if totalSessions === 0, grey out card.

**Research patterns applied:** Pattern 3 (Color-Coded Health Indicators), Research example "Dashboard Overview with Drill-Down".
