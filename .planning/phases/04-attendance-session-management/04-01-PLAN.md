# Phase 4 Attendance Session Management: Implementation Plan

## Objective
Build the Attendance Session Management features. Allow lecturers to create, view, activate, and close attendance sessions for their classes.

## User Review Required
No major breaking changes. The `attendance_sessions` table already exists in the database schema.

## Proposed Changes

### Database & Server Actions
#### [NEW] `src/lib/validations/attendance.ts`
- Zod schema `createSessionSchema` validating `session_date` (string/date), `session_time` (string/time), and `is_retroactive` (boolean).

#### [NEW] `src/lib/actions/attendance.ts`
- `createSession(classId, prevState, formData)`: Validates input and inserts a new session into the `attendance_sessions` table.
- `getSessionsByClass(classId)`: Fetches all sessions for a class ordered by date and time (descending).
- `activateSession(sessionId, classId)`: Updates `is_active` to true and sets `activated_at` timestamp. Triggers `revalidatePath`. Only one session should ideally be active at a time per class, or just update the specific one.
- `closeSession(sessionId, classId)`: Updates `is_active` to false and sets `closed_at` timestamp. Triggers `revalidatePath`.

### Components
#### [NEW] `src/components/features/attendance/CreateSessionForm.tsx`
- A client component using `useActionState` to render a form with Date (HTML date picker), Time (HTML time picker), and Retroactive checkbox fields.
- On success, the form should reset and the page should reflect the new session.

#### [NEW] `src/components/features/attendance/SessionList.tsx`
- A client-side or server-side component that takes an array of sessions.
- Renders each session in a Card or List item format.
- Displays visual indicators for active vs inactive sessions.
- Includes "Activate" and "Close" buttons that trigger the respective server actions.
- Displays activation/closure timestamps if available.

### Pages
#### [MODIFY] `src/app/lecturer/classes/[id]/page.tsx`
- Update the Lecturer Class Detail view to fetch `getSessionsByClass`.
- Add a new section containing the `CreateSessionForm` (perhaps in a Dialog/Modal or a Collapsible section to save screen space).
- Add the `SessionList` component to display the chronological list of sessions below the Create form.

## Verification Plan
### Automated Tests
- Run `npx tsc --noEmit` to verify type safety across all forms and server actions.

### Manual Verification
1. Open the Lecturer Dashboard and navigate to a class.
2. Verify the Create Session form is visible.
3. Submit a new session for today's date and a specific time.
4. Verify the session appears in the Session List as "Inactive".
5. Click "Activate" on the session and verify its status changes to "Active" and the `activated_at` timestamp is set.
6. Click "Close" on the active session and verify its status changes back to "Inactive" and the `closed_at` timestamp is set.
