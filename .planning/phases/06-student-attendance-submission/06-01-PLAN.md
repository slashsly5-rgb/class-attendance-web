# Phase 6 Student Attendance Submission: Implementation Plan

## Objective
Enable students to view their enrolled classes, see pending active attendance sessions, pass location verification (Phase 5), and submit their attendance status.

## User Review Required
No major design decisions. We will use `student_id` (the ID from the `students` table, acquired from a mock login or param for now, since auth is v2) to track the student. Wait, since we don't have authentication yet, how does a student access their dashboard?
For MVP, they will land on a page where they enter their Student ID, which logs them into a mock "session", or we pass their `studentId` as a URL query param. Let's use a very basic form that asks "Enter your Student ID to view dashboard" and redirects to `/student/dashboard?studentId=...`, OR we create a `src/app/student/login/page.tsx` that sets a cookie. A cookie is safer and easier.

## Proposed Changes

### 1. Simple Student Authentication (MVP)
Since we skipped Auth for v1, we need a way for students to "log in" simply to know WHO is submitting attendance.
- **[NEW] `src/app/student/page.tsx`**: A simple form asking for "Student ID".
- **[NEW] `src/lib/actions/auth.ts`**: Action that checks if `students` contains this `student_id`. If so, sets a secure HTTP-only cookie `student_id`.
- **[NEW] `src/lib/utils/auth.ts`**: Helper to read `student_id` cookie.

### 2. Validation & Actions
#### [MODIFY] `src/lib/validations/attendance.ts`
- Add `submitAttendanceSchema` validating `status` (Present, Late, Absent) and `reason` (string, required if Late/Absent).

#### [MODIFY] `src/lib/actions/attendance.ts`
- Add `submitAttendance(sessionId, studentId, formData)`: Validates input, checks if session is still active, ensures `attendance_records` doesn't already have an entry for `(session_id, student_id)`, and inserts the record.
- Add `getStudentEnrollments(studentId)`: Joins `enrollments` and `classes` to list classes the student is in.
- Add `getActiveSessionForClass(classId)`: Returns the single active session for a class, if any.

### 3. Student Dashboard UI
#### [NEW] `src/app/student/dashboard/page.tsx`
- Reads `student_id` cookie.
- Fetches enrolled classes via `getStudentEnrollments`.
- Displays a grid of classes. For each class, checks if there's an active session and shows a "Submit Attendance" alert if so.

#### [NEW] `src/app/student/classes/[id]/page.tsx`
- Class detail page for the student.
- If there's an active session, renders `LocationVerification` wrapper. Inside the wrapper, it renders `SubmissionForm`.
- If no active session, says "No active attendance session at this time."

### 4. Components
#### [NEW] `src/components/features/attendance/SubmissionForm.tsx`
- A client-side form with Radio options (Present, Late, Absent).
- If Late/Absent is selected, shows a text area "Reason".
- Submits to `submitAttendance`.

## Verification Plan
1. Enter a valid Student ID at `/student`.
2. See enrolled classes.
3. Select a class with an active session.
4. Pass Location Verification (using Override or mocked coordinates).
5. Submit as "Late" with a reason.
6. Verify successful state and no duplicate submissions allowed.
